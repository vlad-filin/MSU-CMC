Эксперимент 1: применение ненейросетевых моделей для нижней оценки качества¶

В данном эксперименте будет обучена логистическая регрессия и random forest с 200 деревьями. Была также опробована модель AdaBoost, но точность оказалась немного ниже, чем у random forest при том же количестве деревьев, поэтому в отчете используется только random forest.

logreg = models.LogReg()
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(logreg.parameters(), lr=0.001, weight_decay=0.001)
for epoch in range(10):
    for i, data in enumerate(trainloader, 0):
        inputs, labels = data
        inputs, labels = Variable(inputs), Variable(labels)
        optimizer.zero_grad()
        outputs = logreg(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
correct = 0
total = 0
for data in testloader:
    images, labels = data
    outputs = logreg(Variable(images))
    _, predicted = torch.max(outputs.data, 1)
    total += labels.size(0)
    correct += (predicted == labels).sum()

print('Точность логитической регрессии: %d %%' % (
    100 * correct / total))

Эксперимент 2: обучение и подбор параметров сверточной нейросети¶

В данном эксперименте будут последовательно подобраны следующие параметры: размер ядра свертки ( по равномерной сетке от 2 до 8 с шагом 1), количество фильтров на слое(по равномерной сетке от 5 до 38 с шагом в 3), количество сверточных блоков(1, 2 или 3 блока, при этом количество фильтров и размер ядра свертки у всех будет одинаков). Понятно, что не совсем корректно использовать тестовое множество в качестве валидационного, но так как обучающая выборка достаточно мала (5000), то было принято решение для оценки качества при подборе параметров использовать тестовую выборку.

После каждого подбора будут сделаны выводы о качестве и времени. Для удобства чтения выводы будут продублированы в конце данного эксперимента.

Можно наблюдать снижение точности при увеличении числа сверточных блоков. Возможно, это происходит потому что, параметры второго и третьего сверточного блока не настраивались отдельно, а просто совпадали с параметрами первого сверточного блока. Также возможно, что число параметров слишком велико, чтобы обучиться на такой небольшой выборке. Время обучения увеличивается с ростом числа блоков.

Можно заметить, что наибольшие значения точности достигаются на значениях коэффициента близких к 1 (0.9). При маленьких значениях($10^{-3}$) алгоритм дает результаты схожие с SGD без применения momentum. При этом время обучения менялось незначительно при различных значениях, что логично, так как momentum по сути лишь требует небольших затрат по памяти и не требует трудоемких вычислений. В данном эксперименте будут использованы параметры, подобранные для нейросети ( размер ядра свертки, количество фильтров и тд) так как в пятом эксперименте необходимо использовать автокодировщик в качестве первых слоев сети, следовательно имеет смысл подбирать stride, padding и тд для данного размера ядра свертки, количества фильтров и тд.

В ходе выполнения эксперимента возникли проблемы с памятью, dataloader падал по памяти как только заходил в цикл, поэтому был написан свой dataloader, реализация может быть найдена в models.py. По этой причине время обучения может оказаться больше, чем у остальных студентов.

В данном эксперименте не было сделано особенных выводов, были подобраны параметры для pooling слоя, и был визуализирован выход обученного автокодировщика.
